## Directory Purposes

**Root Level:**
- Purpose: Habit Tracker PWA application
- Contains: HTML entry point, manifest, service worker, main assets
- Key files: `index.html`, `manifest.json`

**css/:**
- Purpose: Styling for Habit Tracker
- Contains: Single stylesheet with layout, responsive design, modal styles
- Key files: `css/styles.css`

**js/:**
- Purpose: Habit Tracker logic (client-side SPA)
- Contains: App controller, data model, view renderers, UI interactions
- Key files: `js/app.js` (orchestrator), `js/data.js` (model)

**icons/:**
- Purpose: PWA icons for installation
- Contains: PNG icons (192x192, 512x512)
- Key files: `icons/icon-192.png`, `icons/icon-512.png`

**newsletter-digest/:**
- Purpose: Backend pipeline for processing newsletters and generating digests
- Contains: Python modules for data extraction, processing, summarization
- Key files: `main.py` (entry point), `config.yaml` (configuration)

**trends-dashboard/:**
- Purpose: Frontend visualization of newsletter trends
- Contains: HTML dashboard, JavaScript chart renderers, data files
- Key files: `trends-dashboard/index.html`, `trends-dashboard/js/app.js`

**trends-dashboard/data/:**
- Purpose: Static JSON data generated by newsletter digest pipeline
- Contains: Weekly digest summaries with structured metadata
- Generated: By `newsletter-digest/trend_extractor.py`
- Committed: Yes (allows dashboard to work without backend)

**trends-dashboard/js/:**
- Purpose: Dashboard visualization logic
- Contains: Chart renderers (Chart.js), data loading, filtering
- Key files: `js/app.js` (orchestrator), `js/data-loader.js` (data fetch)

## Naming Conventions

**Files:**
- **camelCase** for JavaScript files: `daily-view.js`, `grid-view.js`, `habit-editor.js`
- **snake_case** for Python files: `gmail_client.py`, `trend_extractor.py`, `config_loader.py`
- **kebab-case** for data files: `digest-2026-W10-fri.json`
- **lowercase** for style files: `styles.css`, `dashboard.css`

**Directories:**
- **kebab-case** for multi-word directories: `trends-dashboard`, `newsletter-digest`
- **lowercase** for single-word directories: `css`, `js`, `data`, `icons`

**Functions & Variables:**
- **camelCase** in JavaScript: `renderDailyView()`, `toggleCheck()`, `currentMonthKey`
- **snake_case** in Python: `fetch_emails()`, `detect_newsletters()`, `load_config()`
- **UPPERCASE** constants: `MONTH_PREFIX`, `CONFIG_KEY`, `CACHE_NAME`

**Types & Classes:**
- **PascalCase** not used (no class-based code detected)
- Modules export functions and plain objects

## Where to Add New Code

**New Feature in Habit Tracker:**
- Primary code: `js/[feature-name].js`
- If view-related: Modify or create view renderer in `js/*-view.js`
- If data-related: Add queries to `js/data.js`
- Styles: Add to `css/styles.css` with block comment (// Feature Name)
- Example: New workout tracking view → `js/workout-view.js`

**New Section in Trends Dashboard:**
- Implementation: `trends-dashboard/js/[section-name].js`
- Import and render: Add to `trends-dashboard/js/app.js`
- Add container: New `<section>` in `trends-dashboard/index.html`
- Styles: Add to `trends-dashboard/css/dashboard.css`
- Example: New visualization → `trends-dashboard/js/market-share.js`

**New Pipeline Stage in Newsletter Digest:**
- Module: `newsletter-digest/[stage-name].py`
- Export: Function with clear input/output contract
- Integrate: Import in `main.py` and call in pipeline sequence
- Config: Add settings to `config.yaml` if needed
- Example: New summarization approach → `newsletter-digest/ai_summarizer.py`

**Utilities/Helpers:**
- Shared helpers: Create new file in relevant directory
- Habit Tracker helpers: `js/utils.js` (or feature-specific)
- Dashboard helpers: `trends-dashboard/js/utils.js`
- Newsletter helpers: `newsletter-digest/utils.py`
- HTML/DOM escape functions: Already exist in relevant files (escapeHtml, escapeAttr)

**Data Files:**
- Generated data: Place in `trends-dashboard/data/`
- Config files: Place in `newsletter-digest/` root
- Static assets: Place in `css/`, `js/`, `icons/` as appropriate

## Special Directories

**venv/ (in newsletter-digest/):**
- Purpose: Python virtual environment for isolated dependencies
- Generated: Yes (created by `python -m venv venv`)
- Committed: No (listed in .gitignore)
- Usage: Activate with `source venv/bin/activate` (macOS/Linux)

**.planning/codebase/:**
- Purpose: GSD codebase mapping documents (this and ARCHITECTURE.md)
- Generated: Yes (by /gsd:map-codebase command)
- Committed: Yes (referenced by /gsd:plan-phase and /gsd:execute-phase)

**trends-dashboard/data/:**
- Purpose: Weekly trend digests (JSON)
- Generated: Yes (by `newsletter-digest/trend_extractor.py`)
- Committed: Yes (allows offline dashboard operation)
- Versioning: Each week gets new file (digest-2026-W10-fri.json)

**__pycache__/:**
- Purpose: Python bytecode cache
- Generated: Yes (automatic)
- Committed: No (listed in .gitignore)
